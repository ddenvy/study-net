# Потоки ввода-вывода в C#

## 1. Базовые потоки

### FileStream
- Работа с файлами
- Синхронные и асинхронные операции
- Буферизация
- Блокировка файлов

### MemoryStream
- Работа с памятью
- Буферизация в памяти
- Преобразование в массив байтов

### BufferedStream
- Буферизация операций
- Оптимизация производительности
- Работа с другими потоками

## 2. Текстовые потоки

### StreamReader
- Чтение текста
- Кодировки
- Методы:
  - `ReadLine()`
  - `ReadToEnd()`
  - `Read()`

### StreamWriter
- Запись текста
- Кодировки
- Методы:
  - `WriteLine()`
  - `Write()`
  - `Flush()`

## 3. Бинарные потоки

### BinaryReader
- Чтение примитивных типов
- Методы:
  - `ReadInt32()`
  - `ReadString()`
  - `ReadBytes()`

### BinaryWriter
- Запись примитивных типов
- Методы:
  - `Write(int)`
  - `Write(string)`
  - `Write(byte[])`

## 4. Сжатие

### DeflateStream
- Сжатие данных
- Методы:
  - `Compress`
  - `Decompress`

### GZipStream
- Сжатие в формате GZip
- Методы:
  - `Compress`
  - `Decompress`

## 5. Криптография

### CryptoStream
- Шифрование данных
- Работа с криптографическими алгоритмами
- Методы:
  - `Encrypt`
  - `Decrypt`

## 6. Асинхронные операции

### Асинхронные методы
- `ReadAsync()`
- `WriteAsync()`
- `FlushAsync()`
- `CopyToAsync()`

### Отмена операций
- `CancellationToken`
- `CancellationTokenSource`

## 7. Лучшие практики

### Рекомендации
1. Использовать `using` для освобождения ресурсов
2. Правильно обрабатывать исключения
3. Использовать асинхронные операции
4. Закрывать потоки после использования
5. Использовать буферизацию

### Антипаттерны
1. Не закрывать потоки
2. Игнорировать исключения
3. Использовать синхронные операции в UI
4. Не использовать буферизацию
5. Держать потоки открытыми долго
